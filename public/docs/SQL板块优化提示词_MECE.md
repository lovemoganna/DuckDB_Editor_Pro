# SQL 板块 AI 优化提示词

---

## 优化目标

基于 MECE 原则优化 SQL 编辑器模块，实现 **SQL 编写能力的快速调用**、**智能辅助的即时响应**与 **复杂查询的稳定输出**。整体结构需清晰、调用路径明确，确保在复杂场景下稳定输出可执行的 DuckDB SQL 逻辑。

---

## 模块优化要求（适用于 SQL 编辑器各子功能）

### 1. 内嵌「AI 一键填充」功能

**目标**：降低用户 SQL 编写门槛，快速生成上下文相关的查询示例

**实现要求**：
- 每个子功能（智能填充、片段填充、自然语言生成）均需提供 AI 填充入口
- 根据当前模块上下文智能推荐填充内容：
  - **智能填充**：根据用户选中的表和列，生成对应的 SQL 查询草稿
  - **片段填充**：根据选定的分类生成对应业务场景的 SQL 模板
  - **自然语言生成**：根据用户描述的需求，将自然语言转译为 DuckDB SQL
- 填充结果支持二次编辑，确认后一键插入编辑器
- 提供实时预览功能（500ms 防抖），让用户在提交前可预览 AI 生成结果

### 2. 内嵌「快速清除」按钮

**目标**：提升迭代效率，支持一键重置输入状态

**实现要求**：
- 每个输入区域均需提供快速清除入口
- 清除范围应包括：
  - SQL 编辑器当前内容
  - 自然语言描述输入框
  - AI 预览区内容
- 支持快捷键操作（如 Escape 键清除 / Ctrl+Z 撤销）
- 清除后保持面板状态，仅重置输入内容
- 支持撤销功能，可恢复最近一次清除的内容

### 3. 提供模块背景说明与 AI 二次优化引导

**目标**：明确使用场景与常见错误，支持用户与 AI 协作优化

**实现要求**：

#### 3.1 模块背景说明
每个子功能需提供清晰的：
- **定位说明**：该功能解决的问题/场景
- **使用场景**：典型用例列表（3-5 个）
- **常见错误**：易错点提示（2-3 个）
- **最佳实践**：推荐使用方式
- **DuckDB 专有语法**：该场景下的 DuckDB 特色写法

#### 3.2 与 AI 协作引导
在侧边栏信息面板提供：
- **上下文提示**：基于当前功能的预置提示词
- **示例展示**：可点击复制的示例请求
- **二次优化入口**：支持用户基于 AI 生成结果继续优化

## 各子功能详细优化规格

### 3.1 智能填充区（Smart Fill）

| 优化项 | 规格说明 |
|--------|----------|
| AI 填充 | 选中表 → 生成基于表结构的上下文相关 SQL 查询 |
| 快速清除 | 清除编辑器内容 + 预览区内容 |
| 背景说明 | 场景：基于现有表结构快速生成查询；错误：生成 SQL 字段名与实际不符 |

**AI 填充示例**：
```
输入：选中表 "orders"，包含字段 order_id, user_id, amount, created_at
输出：SELECT order_id, user_id, amount, created_at FROM orders LIMIT 100
```

### 3.2 片段库区（Snippet Library）

| 优化项 | 规格说明 |
|--------|----------|
| AI 填充 | 选择分类 → 生成对应业务场景的完整 SQL 模板 |
| 快速清除 | 清除选中的片段分类 + 搜索内容 |
| 背景说明 | 场景：快速插入常用 SQL 模式；错误：片段模板未根据实际表结构调整 |

**AI 填充示例**：
```
分类：窗口函数
输出：SELECT *, ROW_NUMBER() OVER (PARTITION BY col ORDER BY date) AS rn FROM table_name
```

### 3.3 自然语言生成区（Natural Language to SQL）

| 优化项 | 规格说明 |
|--------|----------|
| AI 填充 | 输入业务描述 → 生成对应的 DuckDB SQL 查询 |
| 快速清除 | 清除自然语言输入框 + 预览 SQL |
| 背景说明 | 场景：将业务需求直接转为 SQL；错误：描述中混入了非字段名的中文词汇 |

**AI 填充示例**：
```
输入：「统计每个用户最近的订单总金额」
输出：SELECT user_id, SUM(amount) AS total_amount FROM orders GROUP BY user_id
```

### 3.4 执行计划区（Explain Plan）

| 优化项 | 规格说明 |
|--------|----------|
| AI 填充 | 输入慢查询 → 生成 EXPLAIN ANALYZE 分析 + 优化建议 |
| 快速清除 | 清除执行计划展示内容 |
| 背景说明 | 场景：诊断查询性能问题；错误：只看计划不分析实际执行数据 |

**AI 填充示例**：
```
输入：复杂 JOIN 查询
输出：EXPLAIN ANALYZE 输出 + 优化建议（如下推 WHERE 条件、使用 CTE 物化）
```

### 3.5 数据转换区（Data Transform）

| 优化项 | 规格说明 |
|--------|----------|
| AI 填充 | 描述转换需求 → 生成 PIVOT/UNPIVOT/类型转换 SQL |
| 快速清除 | 清除转换配置 + 预览 SQL |
| 背景说明 | 场景：行列转换、字段清洗；错误：UNPIVOT 时列数据类型不一致 |

**AI 填充示例**：
```
输入：「把 q1/q2/q3/q4 列转为季度维度」
输出：UNPIVOT 语句，将宽表转为长表结构
```

### 3.6 实用工具区（Utilities）

| 优化项 | 规格说明 |
|--------|----------|
| AI 填充 | 描述需求 → 生成测试数据/抽样/质量检查 SQL |
| 快速清除 | 清除工具配置 + 生成的 SQL |
| 背景说明 | 场景：生成测试数据、查看数据摘要；错误：random() 未固定种子导致不可重现 |

**AI 填充示例**：
```
输入：「生成 1000 行用户测试数据，包含姓名、邮箱、注册日期」
输出：generate_series + random() 组合的 INSERT 语句
```

## 交互设计规范

### 按钮位置
- **AI 填充按钮**：位于 SQL 编辑器顶部工具栏，与「执行」「格式化」按钮并列
- **快速清除按钮**：位于输入框内部，文本为空时隐藏

### 视觉反馈
- AI 填充进行中：显示加载动画 + 「生成中...」提示
- 填充完成：弹窗展示结果，支持「采用」「编辑」「取消」
- 清除操作：瞬间完成，无需动画

### 快捷键支持

| 操作 | 快捷键 |
|------|--------|
| 触发 AI 填充 | Ctrl + Shift + A |
| 快速清除 | Escape |
| 撤销清除 | Ctrl + Z |
| 确认采用 | Enter |

## 与 AI 二次优化的对话示例

### 示例 1：智能填充
```
用户：帮我生成一个基于 orders 表的月度销售统计查询
AI：→ 生成包含 date_trunc + GROUP BY 的月度聚合查询

用户：再改成支持筛选指定年份的版本
AI：→ 更新查询，增加 WHERE created_at >= '2024-01-01' 条件
```

### 示例 2：自然语言生成
```
用户：统计每个用户的首次购买日期和最近一次购买日期
AI：→ 生成使用 MIN/MAX 窗口函数的查询

用户：改成只统计复购用户（购买次数 > 1）
AI：→ 增加 HAVING COUNT(*) > 1 过滤条件
```

### 示例 3：执行计划优化
```
用户：帮我优化这个查询，运行太慢了
AI：→ 解析 EXPLAIN ANALYZE 输出，定位瓶颈节点

用户：能详细解释一下为什么这个 JOIN 这么慢吗
AI：→ 提供具体的优化建议，如下推 WHERE 条件、使用小表驱动
```

## 验收标准

1. ✅ 每个子功能均提供 AI 填充入口，填充内容符合模块定位
2. ✅ 每个子功能均提供快速清除功能，一键重置输入状态
3. ✅ 每个子功能均有明确的背景说明（定位、场景、错误、实践）
4. ✅ 侧边栏提供与 AI 协作的预置提示词示例
5. ✅ 整体交互路径清晰，用户可在 3 次点击内完成任意操作
6. ✅ 复杂场景下 AI 生成内容稳定可执行
7. ✅ 支持撤销功能，可恢复最近一次清除的内容

## 实施优先级

| 优先级 | 模块 | 理由 |
|--------|------|------|
| P0 | 自然语言生成 | 核心亮点功能，直接将需求转为 SQL |
| P0 | 智能填充 | 高频使用，基于表结构生成查询价值大 |
| P1 | 片段库 | 辅助查询，降低常用 SQL 模式编写门槛 |
| P1 | 执行计划 | 性能优化场景，帮助用户定位慢查询 |
| P2 | 数据转换 | 特定场景使用，频率相对较低 |
| P2 | 实用工具 | 辅助性功能，主要用于测试和数据探索 |
