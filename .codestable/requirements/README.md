# requirements — 能力愿景

> 本目录存放产品的能力愿景文档，描述"这个产品要解决什么问题、为谁服务"。

## 目录结构

每个能力愿景一个目录：

```
requirements/
├── {capability-name}/
│   ├── README.md    # 能力愿景概述
│   └── context.md  # 背景和用户故事
└── README.md       # 本文件
```

## 已知的核心能力愿景

### 1. 本体论知识管理（Ontology Module）

**愿景**：让用户用"对象-关系-行动"的方式组织和管理知识，同时支持 AI 辅助建模。

**目标用户**：
- 个人用户：管理人生/工作/学习的知识体系
- 开发者：作为数据库之上的人造知识层

**核心价值**：
- 从原始数据到结构化知识的认知跃迁
- AI 辅助降低建模门槛

**当前状态**：原型，功能基本可用但体验欠佳。

### 2. AI 辅助 SQL 生成（Skills 系统）

**愿景**：让用户用自然语言描述需求，AI 生成正确的 SQL。

**目标用户**：需要写 SQL 但不熟悉语法的用户。

**核心价值**：降低 SQL 学习门槛，加速数据分析。

**当前状态**：可用，支持多种 SQL 操作类型。

### 3. 数据质量管理（Abstraction Lab）

**愿景**：帮助用户理解数据质量，发现异常和因果关系。

**目标用户**：需要对数据进行质量检查和深度分析的用户。

**当前状态**：可用。
