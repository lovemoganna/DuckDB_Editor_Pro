# Ontology 模块重构设计方案

**日期**: 2026-05-07
**状态**: 待用户审核
**版本**: v1.0

---

## 背景与目标

### 现状问题

Ontology 模块目前是一个"预制菜"式的 SQL 教学工具：
- 数据模型硬编码为"我的人生"（4个对象、4条关系）
- 26个 SQL 模板本质是 DuckDB 语法演示，而非本体论应用
- 五层 MECE 分类对普通用户无意义且难以填充
- 无任何机制读取用户真实的数据库 Schema
- "洞察"和"沉思"是空装饰，无实际逻辑

### 重构目标

将 Ontology 打造成 **Schema-to-Ontology 桥接工具**：
1. 自动读取用户 DuckDB 中的真实表结构
2. 自动推断表→对象类型、表间关系→语义连接的映射
3. 用图谱可视化的方式呈现本体结构
4. 支持用户手动/AI 增强语义关系
5. 保留"我的人生"作为内置示例/引导，不做默认内容

### 核心定位

> Ontology = **Schema 理解工具** + **本体构建工具** + **关系可视化工具**

---

## 用户场景

| 场景 | 描述 |
|---|---|
| A. 理解陌生数据库 | 面对一个陌生的数据库时，快速通过图谱理解表结构和关系 |
| B. 梳理关系网络 | 理解"什么影响什么"的关系，用本体论方式组织领域知识 |
| C. 管理结构化知识 | 用本体模型管理笔记/项目/任务，可导出为 DuckDB 表 |

## 核心设计

### 1. 入口重构

**左侧抽屉 Tab 重构（新增两个 Tab）**：

| Tab | 用途 | 图标 |
|---|---|---|
| 我的 Schema | 显示当前 DuckDB 所有表，支持勾选生成图谱 | Database |
| 实体库（CRUD） | 管理本体对象、关系、行动 | List |
| 示例 | "我的人生"可交互引导示例 | Sparkles |

**默认行为**：
- 新用户首次打开 → 自动展示"示例"Tab，带引导流程
- 有数据库的用户 → 默认展示"我的 Schema"Tab
- 已有本体数据的用户 → 默认展示"实体库"Tab

### 2. 三视图重构

| 视图 | 用途 | 数据来源 |
|---|---|---|
| **Schema 图谱**（新增） | 自动生成的表结构可视化，聚焦"有哪些东西" | 用户 DuckDB Schema |
| **本体图谱**（原 graph） | 用户手动/AI 增强的语义关系，聚焦"什么影响什么" | 本体论表 |
| **画布**（原 canvas） | 自由整理，聚焦"如何组织我的认知结构" | Canvas 状态表 |

### 3. Schema → 本体 自动推断引擎

**推断规则**：

| Schema 层 | 本体论层 | 推断逻辑 |
|---|---|---|
| 表名（单数化） | 对象类型 | `orders` → `Order` |
| 外键列（如 `user_id`） | 关系 | 作为 Link 指向 `users` 表 |
| 列名含 `type/status/category` | 属性枚举 | 推断为 Link Type |
| 表名含 `_rel/_link/junction` | 关系表 | 直接映射为 Links |
| 列名含 `parent/ref/superior` | 自引用 | 递归关系 |
| 主键 | 对象实例 | 表中每行 → Ontology Object |

**推断结果展示**：
- 推断后的本体预览（对象类型列表 + 关系列表）
- 用户可编辑、确认后再导入

### 4. "我的人生"示例保留方案

**定位**：内置可交互引导示例，不做默认内容

**实现**：
- 独立 Tab 承载，默认折叠
- 改为三步引导流程：
  1. 展示图谱效果（只读）
  2. 解释本体论基本概念（简短）
  3. CTA："开始构建自己的本体" → 跳转到"我的 Schema"
- 示例数据仍然可用作"一键填充"的演示

### 5. AI 能力重新定位

**原来**：AI 用来生成"通用套路"的本体（无意义）
**改为**：AI 用来理解用户表结构，生成 Schema→本体 推断建议

| 场景 | AI 作用 |
|---|---|
| Schema 推断 | 分析表名、列名、数据内容，推荐对象类型和关系类型 |
| 关系命名 | 基于外键列名，推荐更语义化的关系名称 |
| 本体验证 | 检查本体结构是否有环、孤立节点等 |

### 6. 本体持久化架构

**保留现有 5 表结构**（无需改变）：
- `life_object_type` / `life_object` / `life_link_type` / `life_link` / `life_action`

**命名改为通用**（可选 UI 层面改动）：
- 表名可配置，支持多套本体（如 `project_ontology_object`）
- 左侧抽屉 Schema Tab 支持切换"挂载到哪个本体"

## 组件变更

### 新增组件

| 组件 | 职责 |
|---|---|
| `SchemaTab.tsx` | 左侧抽屉 Tab：展示 DuckDB 表列表，支持勾选 |
| `SchemaGraphView.tsx` | Schema 图谱视图：基于表结构自动生成力导向图 |
| `SchemaInferencePanel.tsx` | 推断预览面板：显示自动推断的本体结构，用户可编辑 |
| `ExampleGuide.tsx` | "我的人生"三步引导流程 |

### 修改组件

| 组件 | 变更 |
|---|---|
| `OntologyPanel.tsx` | Tab 栏增加 Schema Tab；DRAWER_TABS 重构 |
| `D3GraphView.tsx` | 改为显示"本体图谱"，数据源为 Ontology 5表 |
| `OntologyModelingWizard.tsx` | 改为 Schema 推断向导，而非通用概念生成 |

### 删除/废弃

- `ontologyTemplates.ts` 的 26 个 SQL 模板（替换为 Schema 推断功能）
- `data/ontologyTemplates.ts` 的 setup/query/modify/export 分类模板
- "预设库"Tab（完全移除）
- 硬编码的"我的人生"作为默认数据的逻辑

---

## 实现优先级

### Phase 1（最小可行）
1. 新增"我的 Schema"Tab + SchemaGraphView
2. Schema → 本体推断引擎
3. 保留"我的人生"示例 Tab

### Phase 2（功能完善）
4. 本体图谱（D3GraphView）数据源切换为推断后的本体
5. CRUD 支持 Schema 推断结果导入
6. AI 推断增强

### Phase 3（体验优化）
7. 画布视图与本体图谱数据同步
8. 本体导出为 DuckDB 表
9. 多本体支持（命名空间）

---

## 回滚方案

- 所有改动在 `components/Library/` 内完成，不影响其他模块
- Schema Tab 默认折叠，不影响现有用户体验
- "我的人生"示例数据保留在 `data/ontologyDataModel.ts`，不做删除
- 旧 SQL 模板文件标记为废弃（deprecated），保留一份备份在 `data/ontologyTemplates.deprecated.ts`

---

## 验收标准

1. 用户打开 Ontology → 看到"示例"引导 Tab，可交互体验
2. 用户点击"我的 Schema" → 看到 DuckDB 中所有表列表
3. 用户勾选表 → 自动生成 Schema 图谱
4. 用户点击"推断本体" → 看到自动推断的对象类型和关系列表，可编辑确认
5. 用户确认导入 → 数据写入 Ontology 5表 → 本体图谱显示结果
6. "预设库"Tab 完全移除，不干扰用户
