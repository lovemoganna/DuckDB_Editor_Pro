# Library 板块 AI Skills 优化提示词

---

## 优化目标

基于 MECE 原则优化 Library 知识库模块，实现**指定能力的快速调用**、**知识的结构化沉淀**与**即时生成 SQL 模拟方案**。整体结构需清晰、调用路径明确，确保在复杂场景下稳定输出可复用的 SQL 逻辑。

---

## 模块优化要求（适用于 Library 各子模块）

### 1. 内嵌「AI 一键填充」功能

**目标**：降低用户输入门槛，快速生成示例或启发性内容

**实现要求**：
- 每个子模块（速查卡片、SQL模板、学习路径、我的收藏、本体论、数据抽象表）均需提供 AI 填充入口
- 根据当前模块上下文智能推荐填充内容：
  - **速查卡片**：根据用户描述的函数/语法生成精简的速查内容
  - **SQL 模板**：根据选定的分类生成对应业务场景的模板
  - **学习路径**：根据用户技能水平推荐学习节点
  - **我的收藏**：根据用户分析需求推荐常用片段
  - **本体论**：根据业务领域生成概念-属性-关系模型
  - **数据抽象表**：根据用户需求生成 MECE 层级的 SQL 模拟方案
- 填充结果支持二次编辑，确认后一键入库

### 2. 内嵌「快速清除」按钮

**目标**：提升迭代效率，支持一键重置输入状态

**实现要求**：
- 每个表单面板均需提供快速清除入口
- 清除范围应包括：
  - 当前输入框内容
  - 选中的分类/标签
  - 展开的详情面板
- 支持快捷键操作（如 Escape 键清除焦点内容）
- 清除后保持面板状态，仅重置输入内容

### 3. 提供模块背景说明与 AI 二次优化引导

**目标**：明确使用场景与常见错误，支持用户与 AI 协作优化

**实现要求**：

#### 3.1 模块背景说明
每个子模块需提供清晰的：
- **定位说明**：该模块解决的问题/场景
- **使用场景**：典型用例列表（3-5 个）
- **常见错误**：易错点提示（2-3 个）
- **最佳实践**：推荐使用方式

#### 3.2 与 AI 协作引导
在右侧信息面板提供：
- **上下文提示**：基于当前模块的预置提示词
- **示例展示**：可点击复制的示例请求
- **二次优化入口**：支持用户基于 AI 生成结果继续优化

## 各子模块详细优化规格

### 3.1 速查卡片区（Reference Cards）

| 优化项 | 规格说明 |
|--------|----------|
| AI 填充 | 输入函数名/描述 → 生成语法格式、参数说明、示例代码 |
| 快速清除 | 清除搜索框 + 展开的卡片详情 |
| 背景说明 | 场景：日常查询检索语法；错误：内容过于零散 |

**AI 填充示例**：
```
输入：「日期处理」
输出：DATE_TRUNC、DATE_ADD、DATEDIFF 等函数速查卡片
```

### 3.2 SQL 模板库（SQL Templates）

| 优化项 | 规格说明 |
|--------|----------|
| AI 填充 | 选择分类 → 生成对应业务场景的完整 SQL 模板 |
| 快速清除 | 清除分类筛选 + 新增模板表单内容 |
| 背景说明 | 场景：快速生成业务查询；错误：未检查表名匹配 |

**AI 填充示例**：
```
分类：用户分层
输出：RFM 模型用户分层 SQL 模板（含参数占位符）
```

### 3.3 学习路径区（Learning Path）

| 优化项 | 规格说明 |
|--------|----------|
| AI 填充 | 输入目标/水平 → 生成定制化学习路径 |
| 快速清除 | 清除搜索/筛选状态 |
| 背景说明 | 场景：系统学习技能；错误：零散查阅无序 |

**AI 填充示例**：
```
输入：「7 天速成窗口函数」
输出：Day 1-7 详细学习节点，每个节点含教程+实战
```

### 3.4 个人收藏区（My Snippets）

| 优化项 | 规格说明 |
|--------|----------|
| AI 填充 | 输入业务需求 → 推荐常用 SQL 片段 |
| 快速清除 | 清除收藏夹筛选 + 搜索内容 |
| 背景说明 | 场景：归档常用查询；错误：无分类无整理 |

**AI 填充示例**：
```
输入：「电商数据分析常用」
输出：订单统计、用户留存、转化率等片段推荐
```

### 3.5 本体论区（Ontology）

| 优化项 | 规格说明 |
|--------|----------|
| AI 填充 | 输入业务领域 → 生成概念-属性-关系模型 |
| 快速清除 | 清除展开的视图/条目详情 |
| 背景说明 | 场景：建立数据认知框架；错误：层级混乱关系不清 |

**AI 填充示例**：
```
输入：「电商订单领域」
输出：订单概念、属性（订单号、金额、状态）、关系（用户-订单、商品-订单）
```

### 3.6 数据抽象表（Abstraction Table）

| 优化项 | 规格说明 |
|--------|----------|
| AI 填充 | 描述需求 → 生成 MECE 层级的 SQL 模拟方案 |
| 快速清除 | 清除展开的抽象表详情 + 参数配置 |
| 背景说明 | 场景：按层级快速调用；错误：表名字段与实际不符 |

**AI 填充示例**：
```
输入：「日活跃用户统计」
输出：
- 概念：用户活跃
- 属性：访问日期、用户 ID
- 关系：用户 → 活跃记录
- 实例 SQL：DAU 统计完整查询（含参数模板）
```

## 交互设计规范

### 按钮位置
- **AI 填充按钮**：位于各子模块顶部操作栏，与「添加」按钮并列
- **快速清除按钮**：位于搜索框/表单内部，文本为空时隐藏

### 视觉反馈
- AI 填充进行中：显示加载动画 + 「生成中...」提示
- 填充完成：弹窗展示结果，支持「采用」「编辑」「取消」
- 清除操作：瞬间完成，无需动画

### 快捷键支持
| 操作 | 快捷键 |
|------|--------|
| 触发 AI 填充 | Ctrl + Shift + A |
| 快速清除 | Escape |
| 确认采用 | Enter |

## 与 AI 二次优化的对话示例

### 示例 1：速查卡片
```
用户：帮我生成一个 Redis 缓存穿透防护的 SQL 速查卡片
AI：→ 生成包含 3 个关键函数（SETEX、GETEX、BITSET）的速查卡片

用户：再细化一下，加上击穿场景的示例
AI：→ 更新卡片，增加「缓存击穿」场景的 SQL 示例
```

### 示例 2：数据抽象表
```
用户：帮我设计一个「用户复购率分析」的数据抽象表
AI：→ 生成 MECE 层级结构：
      - 概念：用户复购
      - 属性：首次购买日期、复购间隔、复购次数
      - 关系：用户 → 订单（多对多）
      - 实例 SQL：复购率计算完整查询

用户：改成支持自定义时间窗口的版本
AI：→ 更新 SQL 模板，增加时间窗口参数 ${day_range}
```

## 验收标准

1. ✅ 每个子模块均提供 AI 填充入口，填充内容符合模块定位
2. ✅ 每个子模块均提供快速清除功能，一键重置输入状态
3. ✅ 每个子模块均有明确的背景说明（定位、场景、错误、实践）
4. ✅ 右侧信息面板提供与 AI 协作的预置提示词示例
5. ✅ 整体交互路径清晰，用户可在 3 次点击内完成任意操作
6. ✅ 复杂场景下 AI 生成内容稳定可执行

## 实施优先级

| 优先级 | 模块 | 理由 |
|--------|------|------|
| P0 | 数据抽象表 | 核心亮点功能，需确保 SQL 生成能力 |
| P0 | SQL 模板库 | 高频使用，AI 填充价值大 |
| P1 | 速查卡片 | 辅助查询，AI 填充可降低门槛 |
| P1 | 本体论 | 数据抽象基础，与数据抽象表联动 |
| P2 | 学习路径 | 使用频率相对较低 |
| P2 | 个人收藏 | 主要是个人数据管理 |
