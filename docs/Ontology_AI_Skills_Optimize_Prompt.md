# Ontology 模块 AI Skills 优化提示词（MECE 体系）

---

## 优化目标

基于 MECE 原则系统性优化 Ontology 本体论模块，实现**业务概念的快速建模**、**关系网络的结构化表达**与**即时生成 SQL 模拟方案**。整体结构需清晰、调用路径明确，确保在复杂场景下稳定输出可执行的本体论建模 SQL。

---

## 模块现状概览

Ontology 模块当前包含以下子模块与视图：

| 维度 | 子模块 |
|------|--------|
| **MECE 五大层级** | 基础层（Foundation/DDL）、关系层（Relations/DML）、方法论层（Methodology/DQL）、模式层（Patterns/DQL-Analysis）、领域层（Domains/DDL-Advanced） |
| **三大视图** | Graph（ReactFlow 图谱）、Canvas（自由拖拽画布）、Panel（数据面板） |
| **AI 增强面板** | Insights（AI 图谱分析）、Introspection（日志/反思）、Suggestions（推荐分组） |

---

## 模块优化要求（适用于 Ontology 各子模块）

### 1. 内嵌「AI 一键填充」功能

**目标**：降低用户输入门槛，快速生成概念模型、关系建模或启发性 SQL 内容。

**实现要求**：
- 每个 MECE 层级（5层）和三大视图均需提供 AI 填充入口
- 根据当前上下文智能生成内容：
  - **基础层**：输入对象描述 → 生成概念-属性-关系完整模型（object_type + object 实例）
  - **关系层**：选择关系类型 → 生成 link_type + link 实例 + 权重建模方案
  - **方法论层**：输入业务需求 → 推荐本体论建模范式 + 实施步骤
  - **模式层**：选择模式类型 → 生成递归 CTE、视图、聚合分析等高级 SQL
  - **领域层**：选择/输入领域 → 生成该领域完整概念-关系模型
  - **图谱视图**：输入话题 → AI 生成概念节点布局 + 关系边
  - **画布视图**：输入场景 → AI 生成画布空间组织方案
  - **洞察面板**：基于现有图谱数据 → AI 分析分组建议、聚类异常
- 填充结果支持二次编辑，确认后一键入库或注入 SQL 编辑器
- 支持「智能补全」：基于现有概念推荐相关概念和关系

**AI 填充示例**：

```
场景：基础层 - 用户输入「项目管理」
输出：
- 对象类型：Project (id, name, status, start_date, end_date, budget)
- 对象类型：Task (id, name, priority, assignee_id, project_id)
- 关系：Project has_many Task → link_type PROJECT_CONTAINS
- SQL DDL：CREATE TABLE life_object + INSERT 示例

场景：图谱视图 - 用户输入「家庭财务」
输出：
- 节点：收入、支出、资产、负债、预算
- 边：收入→储蓄（INCREASES）、支出→资产减少（REDUCES）
- 布局：分层树状，左收入、右资产、下支出
```

### 2. 内嵌「快速清除」功能

**目标**：提升迭代效率，支持一键重置输入状态。

**实现要求**：
- 每个面板、表单、AI 填充输入框均需提供快速清除入口
- 清除范围应包括：
  - 当前输入框内容
  - 选中的 MECE 层级/视图模式
  - 展开的详情面板或 SQL 模板
  - 概念/关系的编辑状态
  - 图谱/画布上的选中节点
- 支持快捷键操作（Escape 清除焦点，Ctrl+Shift+O 触发 AI 填充）
- 清除后保持面板状态，仅重置输入内容

### 3. 提供模块背景说明与 AI 二次优化引导

**目标**：明确使用场景与常见错误，支持用户与 AI 协作优化。

**实现要求**：

#### 3.1 模块背景说明
每个子模块需提供清晰的：
- **定位说明**：该模块解决的问题/场景
- **使用场景**：典型用例列表（3-5 个）
- **常见错误**：易错点提示（2-3 个）
- **最佳实践**：推荐使用方式

#### 3.2 与 AI 协作引导
在右侧信息面板提供：
- **上下文提示**：基于当前模块的预置提示词
- **示例展示**：可点击复制的示例请求
- **二次优化入口**：支持用户基于 AI 生成结果继续优化

## MECE 五大层级详细优化规格

### 3.1 基础层（Foundation / DDL）

| 优化项 | 规格说明 |
|--------|----------|
| AI 填充 | 输入概念描述 → 生成 object_type + object 实例完整模型 + 五表 DDL |
| 快速清除 | 清除对象搜索框 + 展开的对象详情 + 选中的 object_type |
| 背景说明 | 场景：从零构建本体论；错误：类型表/实例表顺序颠倒、外键约束冲突 |

**AI 填充示例**：
```
输入：「用户画像」
输出：
- object_type：UserProfile (id, user_id, name, vip_level, created_at)
- object_type：Tag (id, name, category)
- link_type：UserProfile_TAGGED_Tag (many-to-many)
- SQL：CREATE TABLE + INSERT 示例（带 properties JSON）
```

### 3.2 关系层（Relations / DML）

| 优化项 | 规格说明 |
|--------|----------|
| AI 填充 | 选择源/目标对象 → 生成 link_type + link 实例 + 权重方案 |
| 快速清除 | 清除关系筛选 + 选中的 link_type + 展开的关系详情 |
| 背景说明 | 场景：建立实体间关联；错误：关系类型混用（因果/组成混用）、权重归一化缺失 |

**AI 填充示例**：
```
输入：「员工-公司雇佣关系」
输出：
- link_type：EMPLOYED_BY (source: Employee, target: Company, temporal: true)
- 属性：start_date, end_date, position, salary_band
- link 实例示例 + 时态关系权重查询 SQL
```

### 3.3 方法论层（Methodology / DQL）

| 优化项 | 规格说明 |
|--------|----------|
| AI 填充 | 输入业务场景 → 推荐建模方法 + SQL 实施步骤 + 反思模板 |
| 快速清除 | 清除方法论搜索 + 展开的步骤详情 + introspection 输入 |
| 背景说明 | 场景：选择本体论建模范式；错误：方法选择不当（过度抽象/不足抽象）、反思无记录 |

**AI 填充示例**：
```
输入：「遗留系统数据整合」
输出：
- 推荐方法：自底向上 + 混合建模
- 步骤1：盘点现有表 → 识别核心实体
- 步骤2：抽象为 object_type
- 步骤3：映射 link_type + 关系
- 步骤4：生成 DDL + 迁移 SQL
- introspection 模板：每步记录 Q&A 日志
```

### 3.4 模式层（Patterns / DQL-Analysis）

| 优化项 | 规格说明 |
|--------|----------|
| AI 填充 | 选择模式类型 → 生成递归 CTE、视图、物化路径等高级 SQL |
| 快速清除 | 清除模式筛选 + 展开的 SQL 详情 + 视图编辑器内容 |
| 背景说明 | 场景：高级关系查询、路径分析、聚合统计；错误：递归深度未设限、权重聚合逻辑错误 |

**AI 填充示例**：
```
输入：「追踪用户地址变更历史」
输出：
- 推荐模式：时态本体模式（valid_from, valid_to）
- DDL：带历史版本的 address 表
- SQL：变更轨迹递归查询 + diff 分析
- 视图：latest_address_view
```

### 3.5 领域层（Domains / DDL-Advanced）

| 优化项 | 规格说明 |
|--------|----------|
| AI 填充 | 选择或输入领域 → 生成该领域完整概念-关系模型 + 种子数据 |
| 快速清除 | 清除领域筛选 + 展开的概念详情 + seed 数据表单 |
| 背景说明 | 场景：行业垂直建模；错误：概念照搬不适配（互联网套医疗概念）、种子数据脱离实际 |

**AI 填充示例**：
```
输入：「家庭财务管理」
输出：
- object_type：Income, Expense, Asset, Liability, Budget, Account
- link_type：INCREASES, REDUCES, FUNDS, BUDGETS_FOR, BELONGS_TO
- SQL：完整五表 + 12 条 seed INSERT
- 视图：monthly_balance_sheet_view, cash_flow_view
- 初始 introspection：Q1 收支目标是什么？
```

## 三大视图详细优化规格

### 4.1 Graph 视图（ReactFlow 图谱）

| 优化项 | 规格说明 |
|--------|----------|
| AI 填充 | 输入话题 → 生成节点布局方案 + 节点/边数据 + 布局算法建议 |
| 快速清除 | 清除节点搜索 + 取消选中节点 + 收起详情面板 |
| 背景说明 | 场景：全局关系可视化；错误：节点过多堆叠（>100）、边权重未归一化 |

**AI 填充示例**：
```
输入：「家庭关系 + 财务流向」
输出：
- 节点：Father(人), Mother(人), Son(人), Bank(账户), Salary(收入)
- 边：Father --FUNDS--> Bank --SUPPORTS--> Son
- 布局：力导向布局 + 节点大小按 degree 缩放
```

### 4.2 Canvas 视图（自由拖拽画布）

| 优化项 | 规格说明 |
|--------|----------|
| AI 填充 | 输入场景 → 生成空间组织方案（Space/Group/Item 布局） |
| 快速清除 | 清除画布搜索 + 取消选中节点 + 恢复默认视图 |
| 背景说明 | 场景：自由空间规划、头脑风暴；错误：空间命名混乱、未利用 Group 层级 |

**AI 填充示例**：
```
输入：「项目管理工作区」
输出：
- Space：工作（蓝色背景）
- Group：项目A、项目B
- Item：里程碑1、任务清单、风险日志
- 布局建议：时间线横向排列
```

### 4.3 Panel 视图（数据 CRUD 面板）

| 优化项 | 规格说明 |
|--------|----------|
| AI 填充 | 输入描述 → 生成 object/link/action CRUD 表单预填内容 |
| 快速清除 | 清除搜索 + 取消选中行 + 折叠详情面板 |
| 背景说明 | 场景：批量数据管理；错误：JSON properties 格式错误、link weight 超范围 |

## AI 增强面板详细优化规格

### 5.1 Insights 洞察面板（Graph Analysis）

| 优化项 | 规格说明 |
|--------|----------|
| AI 填充 | 基于当前图谱数据 → AI 分析分组建议、类型聚类、异常检测 |
| 快速清除 | 清除洞察筛选 + 收起展开的分析卡片 |
| 背景说明 | 场景：图谱质量评估；错误：聚类阈值不当（过粗/过细）、孤立节点未处理 |

### 5.2 Introspection 日志面板（Journal/Q&A）

| 优化项 | 规格说明 |
|--------|----------|
| AI 填充 | 输入反思主题 → AI 生成引导性问题 + 记录模板 |
| 快速清除 | 清除日志搜索 + 收起 Q&A 展开项 |
| 背景说明 | 场景：定期自我反思；错误：无结构泛泛而谈、未关联本体概念 |

### 5.3 Suggestions 推荐面板（AI 智能推荐）

| 优化项 | 规格说明 |
|--------|----------|
| AI 填充 | 基于现有数据 → AI 推荐可添加的概念、关系、行动 |
| 快速清除 | 清除推荐筛选 + 收起已采纳的推荐 |
| 背景说明 | 场景：本体论补全；错误：推荐过于泛化未结合具体上下文 |

## 交互设计规范

### 按钮位置
- **AI 填充按钮**：位于各 MECE 层级操作栏、三大视图顶部工具栏、AI 增强面板顶部
- **快速清除按钮**：位于搜索框/表单内部，文本为空时隐藏
- **二次优化入口**：位于 AI 生成结果面板底部（"再细化一下..."输入框）

### 视觉反馈
- AI 填充进行中：显示加载动画 + 「AI 构思中...」提示（带终止按钮）
- 填充完成：弹窗展示结果，支持「采用并入库」「编辑后再用」「取消」
- 清除操作：瞬间完成，无需动画

### 快捷键支持

| 操作 | 快捷键 |
|------|--------|
| 触发 AI 填充 | Ctrl + Shift + O |
| 快速清除 | Escape |
| 确认采用 | Enter |
| 切换视图模式 | Ctrl + 1/2/3（Graph/Canvas/Panel） |
| 切换 MECE 层级 | Ctrl + [ / Ctrl + ] |
| 打开 Insights | Ctrl + I |

## 与 AI 二次优化的对话示例

### 示例 1：基础层概念建模

```
用户：帮我生成一个「产品」概念的本体模型
AI：→ 生成 Product object_type + 基础属性集合 + 关联关系

用户：再细化一下，加上价格相关的维度属性和时间版本
AI：→ 增加 price, discount, currency 字段 + valid_from/to 时态字段
```

### 示例 2：图谱视图关系可视化

```
用户：帮我可视化一个「供应链」领域的概念图谱
AI：→ 生成节点（供应商、原料、工厂、物流、客户）+ 边（供应、生产、运输、销售）
      + 力导向布局参数 + 节点颜色按类型映射

用户：加上异常检测，哪些节点度数异常高或孤立
AI：→ 追加 analytics 视图，标记异常节点 + degree 分布直方图
```

### 示例 3：领域层垂直建模

```
用户：帮我设计一个「金融风控」领域的本体模型
AI：→ 生成 Account, Transaction, Merchant, Device, Anomaly object_type
      + 核心关系定义
      + 欺诈检测 SQL（交易金额异常、IP 聚集、设备指纹）
      + 种子数据 10 条

用户：改成支持实时流式计算的时间窗口版本
AI：→ 更新 SQL 模板，将时间窗口参数化 ${start_ts} ${end_ts}
      + 追加滑动窗口聚合 SQL
```

## 验收标准

1. ✅ 每个 MECE 层级（5层）均提供 AI 填充入口，填充内容符合该层级定位
2. ✅ 三大视图（Graph/Canvas/Panel）均提供 AI 填充功能
3. ✅ AI 增强面板（Insights/Introspection/Suggestions）均提供 AI 协作入口
4. ✅ 每个面板均提供快速清除功能，一键重置输入状态
5. ✅ 每个层级/视图均有明确的背景说明（定位、场景、错误、实践）
6. ✅ 提供与 AI 协作的预置提示词示例 + 可点击复制
7. ✅ 整体交互路径清晰，用户可在 3 次点击内完成任意操作
8. ✅ 复杂场景下 AI 生成内容稳定可执行（JSON 校验 + SQL 语法预检）
9. ✅ AI 生成结果支持二次编辑，确认后一键入库
10. ✅ 快捷键覆盖所有高频操作（AI 填充、清除、视图切换）

## 实施优先级

| 优先级 | 模块 | 理由 |
|--------|------|------|
| P0 | 基础层 AI 填充 | 核心入口，用户使用频率最高 |
| P0 | 领域层 AI 填充 | 高价值场景，直接输出完整领域模型 |
| P0 | Panel 视图 CRUD AI 填充 | 日常数据管理，与 SQL 编辑器联动 |
| P1 | Graph 视图 AI 节点生成 | 图谱可视化增强，AI 构思>手动拖拽 |
| P1 | Insights 面板 AI 分析增强 | 现有启发式算法升级为 LLM 驱动 |
| P1 | 快速清除全模块覆盖 | 迭代效率基础保障 |
| P2 | Canvas 视图 AI 空间布局 | 创意辅助，非核心路径 |
| P2 | Introspection AI 引导 | 辅助反思，使用频率较低 |
| P3 | Suggestions AI 推荐 | 锦上添花，当前启发式已可满足 |
