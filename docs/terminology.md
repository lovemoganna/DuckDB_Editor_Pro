# AI Schema Analysis 模块术语表

## 核心术语定义

| 英文 | 中文 | 缩写 | 定义 |
|------|------|------|------|
| Semantic Type | 语义类型 | - | 列的业务含义分类 (DIM/MEA/TIME/ID/TEXT/CURR/PII) |
| Semantic Annotation | 语义标注 | - | 对列的业务含义进行标记和描述的过程 |
| Quality Score | 质量评分 | - | 数据健康度的量化指标 (0-100) |
| Quality Grade | 质量等级 | - | 质量评分的字母等级 (A/B/C/D) |
| Intent | 分析意图 | - | 用户使用数据的业务目的 (EXPLORATION/AUDIT/REPORTING/FORECAST) |
| Metric | 关键指标 | - | 核心业务度量值，如 revenue, ARPU 等 |
| Scorecard | 指标画像 | - | 包含指标统计信息的详细卡片 |
| Insight | 洞察 | - | AI 发现的数据特征或规律 |
| Hypothesis Insight | 带假设的洞察 | - | 包含前提假设和数据限制的洞察 |
| Reasoning | 推理过程 | - | AI 决策的依据和逻辑链条 |
| Confidence | 置信度 | - | AI 判断的可信程度 |
| Confidence Score | 置信度分数 | - | 0-100 的精确置信度数值 |
| Assertion | 断言 | - | 数据质量验证规则 |
| Custom Assertion | 自定义断言 | - | 用户创建的验证规则 |
| Template | 模板 | - | 可复用的分析配置或 SQL 片段 |
| Pipeline | 流水线 | - | 数据分析的分阶段处理流程 |
| Snapshot | 快照 | - | 特定时间点的数据状态 |

## 语义类型定义

- **DIM (Dimension)**: 维度列 - 用于分组的分类数据
- **MEA (Measure)**: 度量列 - 可聚合的数值数据
- **TIME**: 时间列 - 表示时间维度的列
- **ID**: 标识列 - 主键或唯一标识符
- **TEXT**: 文本列 - 非结构化文本数据
- **CURR**: 金额列 - 货币或价格数据
- **PII**: 敏感列 - 个人身份信息

## 质量等级定义

- **A (90-100)**: 优秀 - 数据质量卓越，无需处理
- **B (75-89)**: 良好 - 数据质量良好，轻微问题可忽略
- **C (60-74)**: 一般 - 存在需要关注的问题
- **D (0-59)**: 较差 - 存在严重质量问题需处理

## 置信度等级

- **High (80-100)**: 高置信度 - AI 判断高度可信，建议采纳
- **Medium (50-79)**: 中等置信度 - 建议人工复核
- **Low (0-49)**: 低置信度 - 需要人工确认
